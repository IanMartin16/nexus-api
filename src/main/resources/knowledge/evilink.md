# Evilink

Evilink es un ecosistema de APIs, servicios y capas de integración orientado a developers y equipos técnicos.

Su enfoque actual es ofrecer productos listos para integrarse, con contratos claros, arquitectura modular y una evolución progresiva hacia una capa de orquestación e inteligencia del ecosistema.

## Productos y piezas principales del ecosistema

### Curpify
API enfocada en validación estructurada en México.

Casos principales:
- Validación de CURP
- Validación de RFC con dígito verificador
- Flujos de identidad y validación fiscal estructurada

No es una suite completa de KYC, biometría o validación documental general.

---

### CryptoLink
API de datos financieros y cripto.

Capacidades principales:
- Consulta de precios
- Snapshots de mercado
- Tendencias
- Momentum
- Risk flags
- Anomalías
- Market health
- Social pulse

Está orientada a integraciones, análisis y consumo de señales de mercado.

---

### Data_Link
API para procesamiento estructurado de archivos y reducción de datos a gran volumen.

Capacidades principales:
- Deduplicación
- Filtrado
- Limpieza de registros
- Transformación estructurada de archivos

Está orientada a flujos técnicos de limpieza, procesamiento y preparación de datos estructurados.

---

### V-Secrets
API orientada a gestión segura de secretos, keys y protección criptográfica.

Capacidades principales:
- Almacenamiento seguro de secretos
- Gestión de keys
- Protección criptográfica para configuraciones e integraciones sensibles

Su propósito es ofrecer una base segura para servicios, integraciones y futuros flujos del ecosistema.

---

### Status-Hub
Superficie de observabilidad del ecosistema.

Capacidades principales:
- Estado de servicios
- Salud operativa básica
- Visibilidad del estado general del ecosistema

No es una plataforma general de observabilidad empresarial, pero sí una pieza importante para monitoreo y evolución operativa de Evilink.

---

## Rutas y piezas especializadas

### Secure Link
Secure Link es una dirección especializada del ecosistema enfocada en seguridad y evaluación de riesgo.

Actualmente debe entenderse como una ruta especializada, no como una función general abierta del widget.

Se relaciona con:
- evaluación de riesgo
- seguridad
- capacidades futuras orientadas a lectura de señales, protección y análisis especializado

---

## Capa de interacción

### Nexus
Nexus es la capa de interacción humana del ecosistema.

Su función es ayudar a orientar, responder y presentar información sobre productos y capacidades del ecosistema.

Nexus no es el cerebro principal del ecosistema. La orquestación y la evolución de discovery recaen progresivamente en MCP-One.

---

## Capa de orquestación

### MCP-One
MCP-One es la capa de orquestación y discovery del ecosistema.

Su función evolutiva incluye:
- clasificación de requests
- discovery de productos y capabilities
- gobernanza de exposición
- routing
- base para futuras capacidades de análisis y orquestación del ecosistema

---

## Estado actual del ecosistema

Evilink ya no se encuentra en una etapa puramente experimental.

Actualmente:
- Curpify ya tiene un cliente real
- el ecosistema ya muestra señales de tracción temprana
- Data_Link y V-Secrets son candidatos fuertes para el soft launch de junio
- Status-Hub ya cumple un papel funcional real dentro del ecosistema
- Secure Link es una pieza estratégica para la evolución futura de seguridad e inteligencia del ecosistema

## Principio de respuesta

Cuando una solicitud no encaje exactamente con un producto:
- se debe orientar hacia el producto más cercano real
- evitar inventar plataformas más amplias de las que hoy existen
- distinguir entre productos públicos, rutas especializadas y capacidades fuera de alcance

## Fuera de alcance actual

Evilink actualmente no debe presentarse como:
- plataforma general de machine learning
- consultoría de IA a medida
- software factory genérica
- suite completa de KYC
- plataforma general de análisis visual o multimedia

Su enfoque actual sigue siendo APIs, servicios técnicos y capas de integración del ecosistema.
